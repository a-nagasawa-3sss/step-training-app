# STEP学習問題作成ツール（問題.md読み込み + スクリーンショットAI判定）

研修問題の`.md`ファイルを読み込ませると、AI（Gemini/Claude/OpenAI、トップページで選択）がタスク単位のSTEP教材（JSON）に自動変換します。受講生はWebページ上でタスクを1つずつ進め、各タスクの「チェックポイント」でAWSマネジメントコンソールのスクリーンショットをアップロードすると、選択中のAIが判定基準に沿って項目ごとに合否判定します。

## 1. 必要なもの（事前準備）

- **Node.js**（v18以上を推奨）
  - インストール済みか確認する：

    ```
    node -v
    ```

    バージョン番号が表示されればOKです。表示されない・コマンドが無いと言われる場合は、
    [Node.js公式サイト](https://nodejs.org/)からインストールしてください（LTS版でかまいません）。

- **Gemini APIキー**

  #### 取得手順

  1. [Google AI Studio](https://aistudio.google.com/apikey) にアクセスします。
  2. 普段使っているGoogleアカウントでログインします（個人のGmailアカウントでOKです）。
  3. 「Create API key」ボタンを押します。
     - 既存のGoogle Cloudプロジェクトが無い場合は、自動で新規プロジェクトが作成されてキーが発行されます。プロジェクト名などを特に意識する必要はありません。
  4. 発行されたキー（`AIza...`から始まる文字列）をコピーします。
     - このキーは後で`.env`ファイルの`GEMINI_API_KEY`に貼り付けます（手順は「2. セットアップ手順」参照）。
     - キーは他人に共有しないでください。漏れると第三者がそのキーで自分のクレジット（無料枠/課金枠）を消費できてしまいます。

  #### 無料枠について

  - APIキーを発行した直後は「無料枠（Free tier）」で利用が始まり、何もしなければ料金は発生しません。
  - 無料枠には「1日あたりのリクエスト数」などの上限があり、使い切ると`429 Too Many Requests`エラーになります（上限は日付が変わる＝UTC基準でリセットされます。詳細は「4. トラブルシューティング」参照）。
  - このアプリは無料枠の上限が比較的緩いモデル（`gemini-2.5-flash-lite`）に固定しているため、通常の研修利用であれば無料枠の範囲内で収まることがほとんどです。

  #### 課金枠を有効化する場合（無料枠の上限に頻繁に当たる人向け・任意）

  無料枠だけで困らない場合はこの手順は不要です。研修期間中に何度も`429`エラーに当たる場合のみ、以下の手順で必要最小限の範囲で課金を有効化してください。

  Gemini APIの課金には「プリペイド（前払いクレジット購入）」と「ポストペイ（後払い）」の2方式があります。使った分だけ後から請求される後払い方式は便利な反面、想定外の高額請求につながるリスクがあるため、**先に少額のクレジットを購入しておくプリペイド方式がミニマムに使うにはおすすめ**です。購入した金額以上は使えない＝それ以上の請求が発生しないため安心です。

  **プリペイド（推奨・少額から試す場合）**

  1. [Google AI Studio Billing](https://aistudio.google.com/billing) のページにアクセスします。
  2. 「Buy credits（クレジットを購入）」を選び、購入金額を指定します（最小2,000円）。今回の用途であればまずは最小で十分試せます。
  3. クレジットカードなどの支払い方法を登録し、購入を確定します。
  4. 購入したクレジットの残高分だけGemini APIが利用できるようになります。**「自動リロード」はOFFのままにしてください。** ONにすると残高が指定額を下回った際に自動で追加購入されてしまうため、手動で都度少額を購入する運用をおすすめします。
  5. 購入したクレジットは購入から1年で失効する点に注意してください（失効分は返金されません）。

  **ポストペイ（使った分だけ後払い・上級者向け）**

  1. [Google AI Studio](https://aistudio.google.com/apikey) の画面で、APIキーが紐づいているプロジェクトの「Billing（請求設定）」から、クレジットカードなどの支払い方法を登録します。
  2. 課金を有効化すると、Pay-as-you-go（使った分だけ支払う）形式に切り替わります。最低料金や月額固定費はなく、無料枠を超えた使用量分のみ課金されます。
  3. 想定外の高額請求を避けるため、Google Cloudコンソールの「お支払い」→「予算とアラート」で、低めの金額（例：500円〜1,000円程度）の予算アラートを設定しておくことを推奨します。アラートはあくまで通知のみで、上限に達しても自動停止はされない点に注意してください。
  4. 研修終了後など課金が不要になったタイミングで、支払い方法の登録解除、または該当プロジェクトの削除を行うと、それ以降の課金は発生しません。

- **Claude（Anthropic）APIキー**（任意・使う場合のみ）

  #### 取得手順

  1. [Anthropic Console](https://console.anthropic.com/) にアクセスし、アカウントを作成（または既存アカウントでログイン）します。
  2. 左側メニューの「API Keys」を開き、「Create Key」を押します。
  3. キー名を入力して発行します。発行されたキー（`sk-ant-...`から始まる文字列）をコピーします。
     - **このキーは発行時にしか表示されません。**コピーし忘れた場合は、新しいキーを発行し直してください。
     - このキーは後で`.env`ファイルの`ANTHROPIC_API_KEY`に貼り付けます（手順は「2. セットアップ手順」参照）。
     - キーは他人に共有しないでください。

  #### 無料枠について（Geminiとの違いに注意）

  - **ClaudeにはGeminiのような恒常的な無料枠はありません。** 新規アカウントに少額の試用クレジットが付与されることはありますが、使い切る・期限切れになる、あるいは支払い方法を一度も登録していない場合、APIの呼び出しはすべて`insufficient_quota`（利用上限到達）のようなエラーで失敗します。
  - そのため、**Claudeを使う場合は基本的に先に少額の課金（クレジット購入）が必要**と考えてください。

  #### 課金（クレジット購入）手順

  1. [Anthropic Console - Billing](https://console.anthropic.com/settings/billing) にアクセスします。
  2. 支払い方法（クレジットカード等）を登録し、クレジットを購入します（少額から購入可能です）。
  3. **「Auto reload（自動リロード）」はOFFのままにしてください。** ONにすると、残高が指定額を下回った際に自動で追加購入されてしまいます。手動で都度少額を購入する運用をおすすめします。
  4. 購入したクレジットの残高分だけClaude APIが利用できるようになります。

- **OpenAI APIキー**（任意・使う場合のみ）

  #### 取得手順

  1. [OpenAI Platform](https://platform.openai.com/api-keys) にアクセスし、アカウントを作成（または既存アカウントでログイン）します。
  2. 「Create new secret key」を押し、キー名を入力して発行します。発行されたキー（`sk-...`または`sk-proj-...`から始まる文字列）をコピーします。
     - **このキーは発行時にしか表示されません。**コピーし忘れた場合は、新しいキーを発行し直してください。
     - このキーは後で`.env`ファイルの`OPENAI_API_KEY`に貼り付けます（手順は「2. セットアップ手順」参照）。
     - キーは他人に共有しないでください。

  #### 無料枠について（Geminiとの違いに注意）

  - **OpenAIにもGeminiのような恒常的な無料枠はありません。** 支払い方法を一度も登録していない・購入したクレジットを使い切っている場合、モデル名に関わらず（`gpt-5-mini`を含むどのモデルでも）APIの呼び出しはすべて`429 insufficient_quota`（利用上限到達）エラーで失敗します。
  - そのため、**OpenAIを使う場合は基本的に先に少額の課金（クレジット購入）が必要**と考えてください（少額の購入で解決することを確認済みです）。

  #### 課金（クレジット購入）手順

  1. [OpenAI Platform - Billing](https://platform.openai.com/settings/organization/billing/overview) にアクセスします。
  2. 支払い方法（クレジットカード等）を登録し、クレジットを購入します（少額（数百円〜）から購入可能です）。
  3. **「Automatic recharge（自動リチャージ）」はOFFのままにしてください。** ONにすると、残高が指定額を下回った際に自動で追加購入されてしまいます。手動で都度少額を購入する運用をおすすめします。
  4. 購入したクレジットの残高分だけOpenAI APIが利用できるようになります。

## 2. セットアップ手順

1. （Windowsの場合）Cドライブ直下に `work` フォルダを作成します（パスは一例です。日本語や空白を含むフォルダパスを避けるための目安です）。

2. ターミナル（コマンドプロンプト/PowerShell/Git Bashなど）で `work` フォルダに移動し、このリポジトリをcloneします。

   ```
   cd C:\work
   git clone -b main https://github.com/a-nagasawa-3sss/step-training-app.git
   ```

   `C:\work\step-training-app` フォルダが作成されれば成功です。続けて、そのフォルダに移動します。

   ```
   cd step-training-app
   ```

3. 依存パッケージをインストールします。

   ```
   npm install
   ```

   `node_modules`フォルダが作成されれば成功です。

4. プロジェクトのルート（`server.js`と同じ階層）に `.env` という名前のファイルを新規作成し、以下の内容を書き込みます。

   ```
   GEMINI_API_KEY=ここに「1. 必要なもの」で取得したGemini APIキーを貼り付ける
   ANTHROPIC_API_KEY=ここにClaude APIキーを貼り付ける（使わない場合は空のままでOK）
   OPENAI_API_KEY=ここにOpenAI APIキーを貼り付ける（使わない場合は空のままでOK）
   PORT=3000
   ```

   - `.env`はリポジトリには含まれていません（`.gitignore`で除外しており、APIキーが他の人と共有・公開されないようにするためです）。各自でこのファイルを作成し、自分のAPIキーを設定してください。
   - `GEMINI_API_KEY`・`ANTHROPIC_API_KEY`・`OPENAI_API_KEY`はどれか1つだけ設定してもかまいません。トップページのAI選択UIでは、APIキーが設定されていないプロバイダーは選択できません。
   - 使用するAIモデルはコスト重視でコード側に固定しています（`.env`側でモデル名を切り替える項目はありません）。
     - Gemini: `gemini-2.5-flash-lite`（無料枠のリクエスト数上限が他モデルより緩いため）
     - Claude: `claude-haiku-4-5`（Claudeのモデルの中で最も低コストなため）
     - OpenAI: `gpt-5-mini`（OpenAIのモデルの中でコストを抑えやすいため。推論コストも抑えるため`reasoning_effort`を最小設定で呼び出します）

5. サーバーを起動します。

   ```
   npm start
   ```

   ターミナルに `http://localhost:3000 で起動しました` と表示されれば成功です。

6. ブラウザで `http://localhost:3000` を開きます。

   問題一覧（ライブラリ）画面が表示されれば準備完了です。

## 3. 使い方

### 3-1. 新しい問題を読み込む（初回 / 新しい教材を追加するとき）

1. トップページの「新しい問題(.md)を読み込む」から、研修問題の`.md`ファイルを選択します。
2. トップページで使用するAI（Gemini/Claude/OpenAI）を選び、「この問題を生成する」を押します。選択したAIが教材を解析してタスク単位のSTEP教材に変換するため、**完了まで数十秒〜1分程度かかることがあります**。生成中はそのままお待ちください。
3. 生成が終わると「『（タイトル）』を生成しました」と表示され、「この問題を開始する」ボタンが現れます。押すとそのまま開始できます（後から一覧の「開始/続行する」からいつでも開けます）。

### 3-2. 問題に取り組む

1. 左のナビゲーションでタスク（STEP）を選びます。
2. タスクのGOAL・要件を確認しながら、実際にAWS上でリソースを構築します。
3. 「チェックポイント」欄に、構築できたことを確認できるAWSコンソールや実装コードなどののスクリーンショットをアップロードします。
   - ファイル選択、または該当エリアをクリックしてフォーカスした状態で `Ctrl+V` での貼り付けに対応しています。
   - 複数枚まとめてアップロードできます。
4. 「スクリーンショットをAIに判定してもらう」を押すと、まだ合格していない判定基準だけが選択中のAIに送られ、判定結果に応じて該当項目にチェックが入ります。
5. チェック状態はサーバー側（`data/progress.json`）に保存されるため、ブラウザのリロードやサーバーの再起動後も保持されます。
6. タスク内のすべての判定基準にチェックが入ると、ナビゲーションにそのタスクの✅が付きます。

### 3-3. 問題を再生成・削除する

- 一覧の「この問題を再生成」から別の`.md`（または修正版の同じ`.md`）を読み込ませて、その問題を上書きできます。**再生成すると、そのコースのチェック済み進捗はリセットされます**（確認ダイアログが出ます）。
- 組み込みコース（最初から入っている「クラウドインフラエンジニア(AWS)実践研修 初級編」）は、保護のため再生成・削除ができません。
- 「削除」を押すと、生成済みコースとその進捗データを削除できます。

## 4. トラブルシューティング

### コース生成・AI判定時にエラーになる（AI側が一時的に混雑している）

問題.mdの生成は教材全文を一度にAIへ送るため、入力が長い・AI側が混雑していると、まれに以下のようなエラーになることがあります。

```
コースの生成に失敗しました: ... [503 Service Unavailable] This model is currently experiencing high demand. ...
```

これは一時的なサーバー混雑が原因で、アプリのバグではありません。サーバー側で複数回の自動リトライは行っていますが、それでも失敗する場合は、以下の手順で再試行してください。

1. ターミナルで `Ctrl + C` を押してサーバーを停止する
2. もう一度 `npm start` でサーバーを起動し直す
3. ブラウザをリロードしてから、もう一度「この問題を生成する」を実行する

それでも失敗が続く場合は、数分〜数十分時間を置いてから再試行してください（AI側の混雑状況によります）。

### 「GEMINI_API_KEY/ANTHROPIC_API_KEY/OPENAI_API_KEY が設定されていません」と表示される

選択中のAIプロバイダーに対応するAPIキーが`.env`に設定されていません。「2. セットアップ手順」の4を参照し、`.env`に該当のAPIキーを設定してから、サーバーを再起動（`Ctrl+C`→`npm start`）してください。

なお、トップページのAI選択UIでは、APIキーが設定されていないプロバイダーはそもそも選択できないようになっています。選択肢に表示されない場合も、まずは`.env`の設定漏れを確認してください。

### Claude/OpenAIで「429」「insufficient_quota」「利用上限」と表示される（課金が必要）

```
コースの生成に失敗しました: 429 You exceeded your current quota, please check your plan and billing details. ...（OpenAIの利用上限に達しているか、支払い方法が未設定です。https://platform.openai.com/account/billing で請求設定を確認してください）
```

**Gemini（無料枠あり）と異なり、ClaudeとOpenAIには恒常的な無料枠が無いため、支払い方法・クレジットを登録していないと、このアプリのプロンプト（教材全文を含む比較的長い入力）を送った時点で利用上限エラーになります。** モデル名を変更しても解決しません（アカウント全体の請求設定の問題のため）。

対処方法：

- Claudeの場合：[Anthropic Console - Billing](https://console.anthropic.com/settings/billing) で支払い方法を登録し、少額のクレジットを購入してください。
- OpenAIの場合：[OpenAI Platform - Billing](https://platform.openai.com/settings/organization/billing/overview) で支払い方法を登録し、少額のクレジットを購入してください（5ドル程度の購入で解決することを確認済みです）。
- どちらも「自動リロード/Automatic recharge」は**OFFのまま**にし、必要な分だけ手動で購入する運用を推奨します（「1. 必要なもの」の各APIキーの節も参照）。

### Gemini APIで「429 Too Many Requests」「quota」と表示される

Gemini APIの無料枠（1日あたりのリクエスト数上限）を使い切っています。日付が変わる（UTC基準）までリセットされません。検証を続けたい場合は、Google AI StudioでそのプロジェクトのGemini APIの課金を有効化してください。

### 「AIの応答が出力上限に達し、途中で切れました」と表示される

タスク数が多い・1タスクあたりの文章量が多い教材は、AIが1回の応答で出せるトークン数（出力上限）を超えてしまい、JSONが不完全な状態で途中で切れることがあります（特にClaude/OpenAIで起こりやすい傾向があります）。

対処方法：

- 問題.mdをタスクの区切りで複数ファイルに分割し、それぞれ別の問題として生成する
- 「この問題を再生成」でもう一度試す（教材の構成によって出力量が変わるため、再試行で収まることもあります）

### 「AIの出力をJSONとして解釈できませんでした」と表示される

AIの応答が、想定しているJSON形式から外れてしまった場合のエラーです（AIの生成内容にはある程度のブレがあるため、まれに発生します）。アプリのバグではなく、再試行で解決することがほとんどです。「この問題を再生成」でもう一度試してください。

### コマンドをコピーしてターミナルに貼り付けるとエラーになる

生成された教材のコマンド（```で囲まれた部分）は改行を保ったまま転記するようAIに指示していますが、教材の構成によっては意図通りに生成されないことがあります。その場合は「この問題を再生成」で生成し直すか、コマンドを1行ずつ確認しながら実行してください。

### エラーメッセージ一覧（原因と対処）

問題生成・AI判定のどちらでも、表示されるエラーは概ね以下のいずれかに当てはまります。

| 表示されるエラー文言（抜粋） | 原因 | 対処 |
|---|---|---|
| `GEMINI_API_KEY が設定されていません` 等 | `.env`に該当のAPIキーが未設定 | `.env`にキーを設定してサーバーを再起動 |
| `429` `insufficient_quota` `利用上限` `billing`（Claude/OpenAI） | アカウントの支払い方法未設定・クレジット不足 | 各社の請求設定ページでクレジットを購入 |
| `429 Too Many Requests` `quota`（Gemini） | Geminiの無料枠（1日あたりの上限）を使い切った | 日付変更(UTC)を待つ、または課金を有効化 |
| `[503 Service Unavailable]` `529` `overloaded` | AI側が一時的に混雑している | サーバーが自動リトライ。失敗時は数分後に再試行 |
| `AIの応答が出力上限に達し、途中で切れました` | 教材が長く、AIの1回の出力トークン上限を超えた | .mdを分割する、または再生成する |
| `AIの出力をJSONとして解釈できませんでした` | AIの応答がJSON形式から外れた（一時的なブレ） | 「この問題を再生成」で再試行 |
| `Gemini/Claude/OpenAI API の呼び出しに失敗しました: ...`（判定時） | スクリーンショット判定時に上記いずれかの原因が発生 | 該当する原因の対処を行う |

## 5. 仕様・構成（簡易設計メモ）

```
step-training-app/
  server.js              Expressサーバー本体
                           - コース生成/取得/再生成/削除API（/api/courses...）
                           - 進捗の取得/保存API（/api/progress...）
                           - スクリーンショット判定API（/api/judge）
  data/steps.json         組み込みコース「aws-level1-default」の教材データ（手作業で作成）
  data/courses/           .mdから生成したコース（1ファイル1コース、ファイル名=コースID）
  data/courses/index.json 生成済みコースの一覧（タイトル等のメタ情報のみ）
  data/progress.json      コースごと・タスクごとの判定基準チェック状態（永続化）
  public/index.html       画面のマークアップ（ライブラリ画面・コース画面の2画面構成）
  public/app.js            フロントエンドの全処理（画面切り替え、API呼び出し、状態管理）
  public/style.css         見た目
```

処理の流れの概要や各関数の役割は、`server.js` / `public/app.js` 内のコメントに詳しく記載しています。設計書を別途作る代わりに、コード自体にその場で読める説明を残す方針にしています。

## 9. AIモデルごとの生成精度の比較（参考）

上記サンプル問題.mdを使い、Gemini / Claude / OpenAIの3モデルでコース生成を行い、元データとの忠実度を比較検証しました。

### 全体構成の比較

| 観点 | Gemini | Claude | OpenAI |
|---|---|---|---|
| ステップ数 | 6（タスク1〜6） | 6（タスク1〜6） | 7（タスク1〜6＋「後片付け」） |
| 元のタスク区切りとの一致 | ◎ 完全一致 | ◎ 完全一致 | △ 「後片付け」見出しも独立ステップ化 |

元データの「演習の流れ」表（タスク1〜6）に対して、Gemini・Claudeは1タスク=1stepで忠実に対応。OpenAIは末尾の「後片付け」セクションも見出しとして独立stepにしており、他2モデルとは粒度が異なります。

### 内容の正確性・忠実度

- **Gemini**: 表の値・コードブロックの転記が正確で、判定基準（criteria）も元データの記載事項に厳密に基づいており、架空の情報を付け加えていない。
- **Claude**: 説明文（「なぜ」を語る導入文）が3モデル中最も丁寧で教育的な質は高いが、タスク6の判定基準で「Application-Stack」「Database-Stack」という、元データに記載のないスタック名を推測で補完している箇所があった。実際の判定で名称が一致せず誤って不合格判定になるリスクがある。
- **OpenAI**: 表の値は概ね正確だが、一部のテーブル（出力Outputsの列挙など）を要約的に圧縮しており、判定基準の項目数もGemini/Claudeより少なく粒度が粗い。

### 総合評価

| | 忠実度 | 説明の充実度 | 判定基準の安全性 | 構成の一致度 |
|---|---|---|---|---|
| Gemini | ◎ | ○ | ◎ | ◎ |
| Claude | ○（一部推測あり） | ◎ | △（架空名称の混入） | ◎ |
| OpenAI | ○ | △ | △（項目数少） | △（step分割が異なる） |

元データの意図への忠実度を重視する場合はGemini、教育的な説明の手厚さを重視する場合はClaude（ただし判定基準の架空項目には要注意）、という傾向が見られました。
