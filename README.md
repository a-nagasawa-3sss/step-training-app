# STEP学習問題作成ツール（問題.md読み込み + スクリーンショットAI判定）

研修問題の`.md`ファイルを読み込ませると、Gemini APIがタスク単位のSTEP教材（JSON）に自動変換します。受講生はWebページ上でタスクを1つずつ進め、各タスクの「チェックポイント」でAWSマネジメントコンソールのスクリーンショットをアップロードすると、Gemini APIが判定基準に沿って項目ごとに合否判定します。

## 1. 必要なもの（事前準備）

- **Node.js**（v18以上を推奨）
  - インストール済みか確認する：

    ```
    node -v
    ```

    バージョン番号が表示されればOKです。表示されない・コマンドが無いと言われる場合は、
    [Node.js公式サイト](https://nodejs.org/)からインストールしてください（LTS版でかまいません）。

- **Gemini APIキー**

  #### 取得手順

  1. [Google AI Studio](https://aistudio.google.com/apikey) にアクセスします。
  2. 普段使っているGoogleアカウントでログインします（個人のGmailアカウントでOKです）。
  3. 「Create API key」ボタンを押します。
     - 既存のGoogle Cloudプロジェクトが無い場合は、自動で新規プロジェクトが作成されてキーが発行されます。プロジェクト名などを特に意識する必要はありません。
  4. 発行されたキー（`AIza...`から始まる文字列）をコピーします。
     - このキーは後で`.env`ファイルの`GEMINI_API_KEY`に貼り付けます（手順は「2. セットアップ手順」参照）。
     - キーは他人に共有しないでください。漏れると第三者がそのキーで自分のクレジット（無料枠/課金枠）を消費できてしまいます。

  #### 無料枠について

  - APIキーを発行した直後は「無料枠（Free tier）」で利用が始まり、何もしなければ料金は発生しません。
  - 無料枠には「1日あたりのリクエスト数」などの上限があり、使い切ると`429 Too Many Requests`エラーになります（上限は日付が変わる＝UTC基準でリセットされます。詳細は「4. トラブルシューティング」参照）。
  - このアプリは無料枠の上限が比較的緩いモデル（`gemini-2.5-flash-lite`）に固定しているため、通常の研修利用であれば無料枠の範囲内で収まることがほとんどです。

  #### 課金枠を有効化する場合（無料枠の上限に頻繁に当たる人向け・任意）

  無料枠だけで困らない場合はこの手順は不要です。研修期間中に何度も`429`エラーに当たる場合のみ、以下の手順で必要最小限の範囲で課金を有効化してください。

  Gemini APIの課金には「プリペイド（前払いクレジット購入）」と「ポストペイ（後払い）」の2方式があります。使った分だけ後から請求される後払い方式は便利な反面、想定外の高額請求につながるリスクがあるため、**先に少額のクレジットを購入しておくプリペイド方式がミニマムに使うにはおすすめ**です。購入した金額以上は使えない＝それ以上の請求が発生しないため安心です。

  **プリペイド（推奨・少額から試す場合）**

  1. [Google AI Studio Billing](https://aistudio.google.com/billing) のページにアクセスします。
  2. 「Buy credits（クレジットを購入）」を選び、購入金額を指定します（最小2,000円）。今回の用途であればまずは最小で十分試せます。
  3. クレジットカードなどの支払い方法を登録し、購入を確定します。
  4. 購入したクレジットの残高分だけGemini APIが利用できるようになります。残高が無くなると再度の購入が必要になりますが、自動で追加請求されることはありません（「自動リロード」を別途オンにした場合を除く）。
  5. 購入したクレジットは購入から1年で失効する点に注意してください（失効分は返金されません）。

  **ポストペイ（使った分だけ後払い・上級者向け）**

  1. [Google AI Studio](https://aistudio.google.com/apikey) の画面で、APIキーが紐づいているプロジェクトの「Billing（請求設定）」から、クレジットカードなどの支払い方法を登録します。
  2. 課金を有効化すると、Pay-as-you-go（使った分だけ支払う）形式に切り替わります。最低料金や月額固定費はなく、無料枠を超えた使用量分のみ課金されます。
  3. 想定外の高額請求を避けるため、Google Cloudコンソールの「お支払い」→「予算とアラート」で、低めの金額（例：500円〜1,000円程度）の予算アラートを設定しておくことを推奨します。アラートはあくまで通知のみで、上限に達しても自動停止はされない点に注意してください。
  4. 研修終了後など課金が不要になったタイミングで、支払い方法の登録解除、または該当プロジェクトの削除を行うと、それ以降の課金は発生しません。

## 2. セットアップ手順

1. （Windowsの場合）Cドライブ直下に `work` フォルダを作成します（パスは一例です。日本語や空白を含むフォルダパスを避けるための目安です）。

2. ターミナル（コマンドプロンプト/PowerShell/Git Bashなど）で `work` フォルダに移動し、このリポジトリをcloneします。

   ```
   cd C:\work
   git clone https://github.com/a-nagasawa-3sss/step-training-app.git
   ```

   `C:\work\step-training-app` フォルダが作成されれば成功です。続けて、そのフォルダに移動します。

   ```
   cd step-training-app
   ```

3. 依存パッケージをインストールします。

   ```
   npm install
   ```

   `node_modules`フォルダが作成されれば成功です。

4. プロジェクトのルート（`server.js`と同じ階層）に `.env` という名前のファイルを新規作成し、以下の内容を書き込みます。

   ```
   GEMINI_API_KEY=ここに「1. 必要なもの」で取得したAPIキーを貼り付ける
   PORT=3000
   ```

   - `.env`はリポジトリには含まれていません（`.gitignore`で除外しており、APIキーが他の人と共有・公開されないようにするためです）。各自でこのファイルを作成し、自分のAPIキーを設定してください。
   - 使用するAIモデルは `gemini-2.5-flash-lite` にコード側で固定しています（無料枠のリクエスト数上限が他モデルより緩いため）。`.env`側でモデルを切り替える項目はありません。

5. サーバーを起動します。

   ```
   npm start
   ```

   ターミナルに `http://localhost:3000 で起動しました` と表示されれば成功です。

6. ブラウザで `http://localhost:3000` を開きます。

   問題一覧（ライブラリ）画面が表示されれば準備完了です。

## 3. 使い方

### 3-1. 新しい問題を読み込む（初回 / 新しい教材を追加するとき）

1. トップページの「新しい問題(.md)を読み込む」から、研修問題の`.md`ファイルを選択します。
2. 「Geminiでこの問題を生成する」を押します。Geminiが教材を解析してタスク単位のSTEP教材に変換するため、**完了まで数十秒〜1分程度かかることがあります**。生成中はそのままお待ちください。
3. 生成が終わると「『（タイトル）』を生成しました」と表示され、「この問題を開始する」ボタンが現れます。押すとそのまま開始できます（後から一覧の「開始/続行する」からいつでも開けます）。

### 3-2. 問題に取り組む

1. 左のナビゲーションでタスク（STEP）を選びます。
2. タスクのGOAL・要件を確認しながら、実際にAWS上でリソースを構築します。
3. 「チェックポイント」欄に、構築できたことを確認できるAWSコンソールのスクリーンショットをアップロードします。
   - ファイル選択、または該当エリアをクリックしてフォーカスした状態で `Ctrl+V` での貼り付けに対応しています。
   - 複数枚まとめてアップロードできます。
4. 「スクリーンショットをAIに判定してもらう」を押すと、まだ合格していない判定基準だけがGeminiに送られ、判定結果に応じて該当項目にチェックが入ります。
5. チェック状態はサーバー側（`data/progress.json`）に保存されるため、ブラウザのリロードやサーバーの再起動後も保持されます。
6. タスク内のすべての判定基準にチェックが入ると、ナビゲーションにそのタスクの✅が付きます。

### 3-3. 問題を再生成・削除する

- 一覧の「この問題を再生成」から別の`.md`（または修正版の同じ`.md`）を読み込ませて、その問題を上書きできます。**再生成すると、そのコースのチェック済み進捗はリセットされます**（確認ダイアログが出ます）。
- 組み込みコース（最初から入っている「クラウドインフラエンジニア(AWS)実践研修 初級編」）は、保護のため再生成・削除ができません。
- 「削除」を押すと、生成済みコースとその進捗データを削除できます。

## 4. トラブルシューティング

### コース生成時にエラーになる（Gemini APIのエラーが表示される）

問題.mdの生成は教材全文を一度にGeminiへ送るため、入力が長い・Gemini側が混雑していると、まれに以下のようなエラーになることがあります。

```
コースの生成に失敗しました: ... [503 Service Unavailable] This model is currently experiencing high demand. ...
```

これは一時的なサーバー混雑が原因で、アプリのバグではありません。サーバー側で複数回の自動リトライは行っていますが、それでも失敗する場合は、以下の手順で再試行してください。

1. ターミナルで `Ctrl + C` を押してサーバーを停止する
2. もう一度 `npm start` でサーバーを起動し直す
3. ブラウザをリロードしてから、もう一度「Geminiでこの問題を生成する」を実行する

それでも失敗が続く場合は、数分〜数十分時間を置いてから再試行してください（Gemini側の混雑状況によります）。

### 「429 Too Many Requests」「quota」と表示される

Gemini APIの無料枠（1日あたりのリクエスト数上限）を使い切っています。日付が変わる（UTC基準）までリセットされません。検証を続けたい場合は、Google AI StudioでそのプロジェクトのGemini APIの課金を有効化してください。

### コマンドをコピーしてターミナルに貼り付けるとエラーになる

生成された教材のコマンド（```で囲まれた部分）は改行を保ったまま転記するようGeminiに指示していますが、教材の構成によっては意図通りに生成されないことがあります。その場合は「この問題を再生成」で生成し直すか、コマンドを1行ずつ確認しながら実行してください。

## 5. 仕様・構成（簡易設計メモ）

```
step-training-app/
  server.js              Expressサーバー本体
                           - コース生成/取得/再生成/削除API（/api/courses...）
                           - 進捗の取得/保存API（/api/progress...）
                           - スクリーンショット判定API（/api/judge）
  data/steps.json         組み込みコース「aws-level1-default」の教材データ（手作業で作成）
  data/courses/           .mdから生成したコース（1ファイル1コース、ファイル名=コースID）
  data/courses/index.json 生成済みコースの一覧（タイトル等のメタ情報のみ）
  data/progress.json      コースごと・タスクごとの判定基準チェック状態（永続化）
  public/index.html       画面のマークアップ（ライブラリ画面・コース画面の2画面構成）
  public/app.js            フロントエンドの全処理（画面切り替え、API呼び出し、状態管理）
  public/style.css         見た目
```

処理の流れの概要や各関数の役割は、`server.js` / `public/app.js` 内のコメントに詳しく記載しています。設計書を別途作る代わりに、コード自体にその場で読める説明を残す方針にしています。

## 6. 動作確認用のテストコード

アプリが起動しない、動作がおかしいなど不具合が疑われる場合は、`test/`配下に用意しているテストコードを実行すると、原因の手がかりが得られることがあります。

```
npm test
```

テスト結果をそのままコピペして長沢に連絡、もしくはご自身で修正していただいても良いです。
エラーが出た場合は、その内容を元に原因を確認してください。


## 7. 注意事項

- アップロードした画像はGemini APIへの判定リクエストにのみ使用し、サーバー側のディスクには保存していません。
- `.env`（APIキー）は絶対にgit等で公開しないでください。
